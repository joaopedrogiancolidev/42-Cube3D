Estado das structs antes de start_cube()
=======================================

Contexto
--------
- Execução: ./game src/sample_maps/valid_arthur.cub
- Ponto de inspeção: logo após parser() retornar 0, antes de start_cube(&cube_data).
- Struct viva nesse ponto: t_cube_data cube_data (na stack de main).

1. t_cube_data
--------------

Campos de configuração
----------------------

no_path, so_path, we_path, ea_path
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Vêm das linhas NO/SO/WE/EA e são validadas/ajustadas para caminhos realmente
abríveis a partir do diretório do .cub.

Para valid_arthur.cub, após validate_texture_path():
- cube_data.no_path -> "src/sample_maps/./textures/north.png"
- cube_data.so_path -> "src/sample_maps/./textures/south.png"
- cube_data.we_path -> "src/sample_maps/./textures/west.png"
- cube_data.ea_path -> "src/sample_maps/./textures/east.png"

(Os prefixos exatos podem variar levemente dependendo da CWD na execução,
mas semanticamente são paths válidos para os PNGs em src/sample_maps/textures/.)

floor_rgb e ceiling_rgb
~~~~~~~~~~~~~~~~~~~~~~~
Das linhas:
- F 90,170,90
- C 255,190,140

depois de parse_rgb_triplet() e transfer_config_to_cube():
- cube_data.floor_rgb[0] = 90
- cube_data.floor_rgb[1] = 170
- cube_data.floor_rgb[2] = 90

- cube_data.ceiling_rgb[0] = 255
- cube_data.ceiling_rgb[1] = 190
- cube_data.ceiling_rgb[2] = 140

Conversão para int **map
------------------------

copy_runtime_map() aplica a regra em cada célula (linha=row, coluna=col):
- se cell == '1' ou cell == ' '  -> cube_data.map[row][col] = 1
- caso contrário ('0', 'N', 'S', 'E', 'W') -> cube_data.map[row][col] = 0

Resultado (cada linha: 41 inteiros 0/1):

Interpretação:
- 1 = parede ou espaço de padding (fora do mapa andável)
- 0 = chão andável (inclui a célula de spawn 'N')

2. t_raycast (dentro de t_cube_data)
------------------------------------

O parser só inicializa alguns campos de t_raycast; o resto continua 0
por causa do ft_memset(cube_data, 0, sizeof(*cube_data)) inicial.

Posição do jogador (spawn)
--------------------------

O caractere 'N' está na linha de mapa y=9 (0-based dentro do bloco do mapa)
e coluna x=20 (0-based), após a normalização.

Em fill_map_row():
- cube_data.raycast.pos_x = col + 0.5 = 20.5
- cube_data.raycast.pos_y = row + 0.5 = 9.5

Orientação inicial (spawn 'N')
------------------------------

set_spawn_orientation(&cube_data->raycast, 'N') faz:
- dir_x   = 0.0
- dir_y   = -1.0
- plane_x = 0.66
- plane_y ajustado em set_raycast_dir_plane:
  - como px != 0.0 nesse caso, plane_y = 0.0

Portanto, ao final do parser:
- cube_data.raycast.pos_x   = 20.5
- cube_data.raycast.pos_y   = 9.5
- cube_data.raycast.dir_x   = 0.0
- cube_data.raycast.dir_y   = -1.0
- cube_data.raycast.plane_x = 0.66
- cube_data.raycast.plane_y = 0.0

E em transfer_config_to_cube():
- cube_data.raycast.old_time = 0

Todos os demais campos de t_raycast ainda estão em 0 nesse momento:
- camera_x, raydir_x, raydir_y
- map_x, map_y
- delta_dist_x, delta_dist_y
- side_dist_x, side_dist_y
- step_x, step_y
- hit, side
- perp_wall_dist
- line_height, draw_start, draw_end
- color
- time, frame_time
- move_speed, rot_speed
- olddir_x, oldplane_x

Esses campos serão preenchidos/configurados mais tarde, dentro de start_cube()
e das funções de raycasting (calculate_raycast, calculate_dda, etc.).

3. Campos gráficos e MLX em t_cube_data
---------------------------------------

Antes de start_cube(), o parser NÃO inicializa nada relativo a janela MLX
ou texturas carregadas; esses campos permanecem zerados/NULL devido ao
ft_memset inicial:

- cube_data.win_width  = 0
- cube_data.win_height = 0

- cube_data.image_cube.image  = NULL
- cube_data.image_cube.width  = 0
- cube_data.image_cube.height = 0

- cube_data.minimap = NULL
- cube_data.player  = NULL

- cube_data.mlx = NULL

- cube_data.tex_no = NULL
- cube_data.tex_so = NULL
- cube_data.tex_we = NULL
- cube_data.tex_ea = NULL

- cube_data.scene_dir = NULL (não é preenchido pelo parser)

4. Outras structs temporárias do parser
---------------------------------------

Durante parser(), também existem:
- t_parser_config cfg
- t_map_grid      map_grid

No final de parser(), no caminho feliz:
- free_parser_config(&cfg) é chamado
- free_map_grid(&map_grid) é chamado

Ou seja, ao retornar para main e antes de start_cube(), somente o conteúdo
copiado para t_cube_data importa para o restante do programa.

5. Resumo
---------

No ponto:

    if (parser(argc, argv, &cube_data) != 0)
        return (1);
    // <-- AQUI (antes de start_cube)

para valid_arthur.cub, temos:

- cube_data configurado com:
  - paths de textura NO/SO/WE/EA válidos
  - floor_rgb = {90, 170, 90}
  - ceiling_rgb = {255, 190, 140}
  - map_width  = 41
  - map_height = 15
  - map[y][x]   como grade de 0/1 (acima)
  - raycast.pos = (20.5, 9.5)
  - raycast.dir = (0.0, -1.0)
  - raycast.plane = (0.66, 0.0)
  - raycast.old_time = 0
- todos os campos relacionados a MLX, imagens, texturas carregadas e
  tamanhos de janela ainda estão em 0/NULL e serão inicializados
  apenas dentro de start_cube().
